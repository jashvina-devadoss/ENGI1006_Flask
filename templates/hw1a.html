Homework 1A: Writing a program, Pseudocode, Boolean logic, Binary numbers

How to write a program:

  1. write pseudocode
  2. follow stylistic guidelines: organize the programs into small, self-contained, reusable chunks of code (functions), using a main() function to execute the program. Use whitespace and comments for readability and interpretability. Remember that the program should always terminate!
  3. develop the logic of solving the problem. Use algorithms to understand how to solve a problem. An algorithm is a series of a finite number of steps which you use to come to an answer to a problem
  4. Manage data. Use data structures to keep track of data.

The major structural components of a program are:

  1. an ordered sequence of steps
  2. values stored in variables by assignment
  3. comparison by boolean logic
  4. control flow through selective execution of sections of the code with conditional statements (if-elseif-else)
  5. sections of the code iteratively executed through looping (for & while loops)
  6. blocks of predefined code, or functions, are executed by calling that code
  7. user input and program output (I/O)
  8. the use of randomness (e.g. for choosing a subset of samples)

Pseudocode

We wrote an algorithm using pseudocode, a step by step plan for code structure that is not specific to any programming language. Writing pseudocode is a useful step before writing a program, because it helps organize the structure and catch bugs, holes in logic, and “edge cases”, e.g. in Homework 1A we wrote pseudocode to calculate when a bank account is depleted based on interest and withdrawal rates— it is important to account for a variety of inputs (or “edge cases”) such that an infinite loop would be avoided if the bank account was actually accruing money rather than being depleted.

Boolean logic

We briefly covered Boolean logic, such that True AND True AND False == False (If any one component is False in an AND statement, the whole statement is False), True OR False or False == False (if any one component is True in an OR statement, the whole statement is True), and NOT True == False.
