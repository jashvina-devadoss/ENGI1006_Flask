Homework 1B: Writing a program, Python style guidelines

We wrote a series of functions to create two programs, each executed by a main function. One program used user input to return data about the user’s name, while the other manipulated elements in lists. We tested our code using a tester file that executed the code and checked the outputs.

We wrote the programs using the Python style guidelines:

  1. Write a header for every file with the name of the author of the file and an overview of the functions in the file.
  2. Import statements for libraries should be at the top of the file, so that imports are global.
  3. Use whitespace to improve readability. Use a space around arithmetic operators, such as =, +, -, *, an empty line between chunks of code, and two lines between each function.
  4. Write docstring comments for every function, describing what the function does. ‘ ‘ ‘ Sample docstring format ‘ ‘ ‘
  5. Limit line length to 79 characters. Spyder has a helpful page break indicating the 79 character mark.
  6. Comment logical blocks of code, so that someone else can understand that block of code just by reading the comment.
  7. Maintain uniform indentation- a single indentation block is defined by four spaces or one tab.
  8. Create descriptive variable and function names that explain the purpose of the variable or function.
  9. Avoid global variables defined outside a function, because they are expensive memory-wise.
  10. Big picture: use uniform, coherent file structure. Each Python file should start with a header, followed by import statements, followed by functions.
